# 💰 ExpenseTracker — Personal Expense Tracker (Flask)

A complete, production-ready personal finance web application built with
**Flask**, following the **MVC** pattern with **Flask Blueprints**,
**SQLAlchemy ORM**, **Flask-Migrate**, **Flask-Login**, and **Flask-WTF**.
Track income and expenses, visualize spending with **Chart.js**, generate
monthly reports, and export to **CSV/PDF** — all in a responsive
**Bootstrap 5** UI with dark mode support

---

## ✨ Features

- **Authentication** — registration, login, logout, hashed passwords
  (Werkzeug), session management via Flask-Login, per-user data isolation
- **Transactions** — add / edit / delete / view income & expense entries
  with amount, category, date, payment method, notes, and optional
  receipt image/PDF upload
- **Categories** — predefined categories (Salary, Food, Transport,
  Shopping, Entertainment, Healthcare, Education, Bills, Investment,
  Others) plus fully custom user-defined categories
- **Dashboard** — total income, total expenses, current balance, this
  month's net, recent transactions, and three interactive charts
- **Reports** — filter by month, year, category, and type; export to
  **CSV** or **PDF**, or open a print-friendly view
- **Charts (Chart.js)** — category-wise expense pie/doughnut chart,
  income vs. expense bar chart (6-month view), spending trend line chart
- **Search, sort, filter, pagination** on the transactions list — by
  keyword, date range, category, payment method, and amount range
- **Profile & Settings** — update personal info, change password,
  toggle dark/light mode (persisted per user)
- **Validation & UX** — Flask-WTF form validation, flash messages,
  CSRF protection, custom 404 / 500 error pages
- **Responsive UI** — mobile-friendly sidebar navigation, Bootstrap 5,
  Bootstrap Icons, dark mode

## 🚀 Installation & Setup

### 1. Enter the project directory
```bash
cd expense_tracker
```

### 2. Create and activate a virtual environment
```bash
python -m venv venv

# macOS / Linux
source venv/bin/activate

# Windows
venv\Scripts\activate
```

### 3. Install dependencies
```bash
pip install -r requirements.txt
```

### 4. Configure environment variables
```bash
cp .env.example .env
# then edit .env and set a real SECRET_KEY
```

### 5. Initialize the database (Flask-Migrate)
```bash
export FLASK_APP=run.py        # Windows (cmd): set FLASK_APP=run.py
flask db init
flask db migrate -m "Initial migration"
flask db upgrade
```
This creates `instance/expense_tracker.db` with all tables. Every new
user automatically receives their own copy of the 10 default categories
upon registration — no seed command required.

### 6. Run the development server
```bash
python run.py
```
Visit **http://localhost:5000** — register an account and start tracking!

### 7. (Optional) Run with a production WSGI server
```bash
pip install gunicorn
gunicorn "run:app" --bind 0.0.0.0:8000
```

## 📊 Key Modules

- **`app/services/report_service.py`** — all dashboard/report aggregation
  queries (totals, monthly summary, category breakdown, income-vs-expense,
  spending trend) live here, separate from route/controller logic.
- **`app/services/export_service.py`** — generates in-memory CSV and PDF
  files (via ReportLab) so nothing is written to disk on export.
- **`app/services/upload_service.py`** — validates and stores receipt
  uploads with UUID-based filenames to avoid collisions/overwrites.

## 🖌 Design Notes

- Primary color: Indigo (`#4f46e5`) with a ledger-inspired monospace
  font for numeric amounts.
- Dark mode is applied via Bootstrap 5's `data-bs-theme` attribute and
  persisted per-user in the database; toggling it fires a small AJAX
  request (CSRF-protected) so the preference survives across sessions.
- All destructive actions (delete transaction/category) require a
  confirmation dialog and are CSRF-protected POST requests.
- Receipts are stored under `app/static/uploads/receipts/` with
  UUID-based filenames; max upload size is 5 MB (configurable in
  `config.py`).

## 🧭 Default Categories

Every new account is seeded with:

**Income:** Salary, Investment
**Expense:** Food, Transport, Shopping, Entertainment, Healthcare,
Education, Bills, Others

Users can add, rename, or delete any category from **Categories** in
the sidebar (a category with existing transactions cannot be deleted
until those transactions are reassigned or removed).
